# Auto Loot PoE Helper — План реализации

> Цель: помощник авто-подбора лута для **Path of Exile 2** на **Python**.
> Подход: **только чтение экрана** (скриншот + компьютерное зрение), без чтения/инъекции памяти игры.
> Фильтр пользователя: `Vladislav Yoshi Perfection.filter` = **NeverSink PoE2 0.10.2, TYPE 4 VERY STRICT**.

---

## ⚠️ Дисклеймер
Сторонняя автоматизация игрового процесса нарушает ToS GGG и может привести к **бану аккаунта**.
Память игры не трогаем (это гарантированный бан + читерская инъекция). Работаем исключительно
по скриншоту экрана и эмуляции мыши/клавиатуры. Риск использования — на стороне пользователя.

## 2.5. Стратегия работы с фильтром (КЛЮЧЕВОЕ)

NeverSink переиспользует фоновые цвета между тирами (напр. вся валюта = белый фон
`255 255 255`, отличается лишь текст/рамка + луч PlayEffect + иконка миникарты).
Поэтому детекция по «родным» цветам подписи неоднозначна.

**Решение:** генерируем и впрыскиваем **override-блок** в зону фильтра
`# !! Waypoint c0.start : "Start - Override ALL rules"` (тело начинается ~строка 127).
Блок матчит выбранные категории (валюта, фрагменты, валуйстоны, юники и т.д.) и красит их
в **один уникальный фоновый цвет-маркер**, которого нет больше нигде в фильтре
(напр. `SetBackgroundColor 255 0 254 255`). Детектор ищет ровно этот цвет → быстро и надёжно.

Модуль `core/filter_patcher.py`:
- [ ] парсит .filter, собирает занятые цвета, выбирает свободный цвет-маркер
- [ ] генерирует override Show-блоки для выбранных категорий (стиль: крупный шрифт, маркер-фон, толстая рамка, опц. PlayEffect/MinimapIcon)
- [ ] вставляет блок после waypoint `c0.start`, делает бэкап оригинала (`*.filter.bak`)
- [ ] кнопка/команда «откатить» (восстановить из бэкапа)

Плюсы: детектор всегда синхронен с тем, что показывает игра; не нужно описывать предметы дважды.
Минус: выбранные категории визуально станут одного цвета (для авто-лута это норма).

Альтернатива (если override не нужен): детекция по **лучам PlayEffect** — у ценного дропа
столб света, селективный сигнал, но сложнее в CV (полупрозрачный градиент). Заложить как опцию.

## 3. Этапы реализации

### Этап 0 — Каркас ✅ цель MVP-основа
- [ ] Структура папок, `requirements.txt`, `README.md`
- [ ] `config_manager.py`: загрузка `default.yaml`, мердж с профилем, сохранение
- [ ] `logger.py`: вывод в консоль + файл, уровни
- [ ] `main.py`: пустой главный цикл с graceful-stop по хоткею

### Этап 1 — Захват экрана + детекция
- [ ] `capture/window.py`: найти окно «Path of Exile 2», получить bbox, проверить что оно активно
- [ ] `capture/screen.py`: захват кадра в bbox (dxcam, фолбэк mss), целевые ~30–60 FPS
- [ ] `vision/color_detector.py`: HSV-маска под цвет фильтра → контуры → центры → список точек
- [ ] Калибровочный режим: показать маску в окне для подбора порогов цвета

### Этап 2 — Ввод
- [ ] `input/mouse.py`: плавное движение к точке (безье/линейно) + клик, рандомные задержки
- [ ] `input/keyboard.py`: регистрация глобальных хоткеев (toggle, профили, выход)
- [ ] Защита: действия только когда окно PoE2 в фокусе

### Этап 3 — Логика подбора (loot_engine)
- [ ] `vision/radius.py`: отсев целей вне радиуса от центра экрана (персонажа)
- [ ] `core/loot_engine.py`:
  - [ ] триггер по хоткею (hold/toggle)
  - [ ] сортировка целей по дистанции + приоритету
  - [ ] очередь, анти-дабл-клик (не кликать одну точку дважды подряд)
  - [ ] кулдаун между кликами
- [ ] Режимы: «один клик по нажатию» / «удерживание собирает всё подряд»

### Этап 4 — Фильтр / правила (filter_patcher)
- [ ] `core/filter_patcher.py`: парсинг NeverSink .filter, выбор свободного цвета-маркера
- [ ] Генерация override Show-блоков для выбранных категорий + вставка после `c0.start`
- [ ] Бэкап оригинала (`*.filter.bak`) + команда отката
- [ ] `core/rules.py`: какие категории включены, приоритеты (валюта > фрагменты > валуйстоны > прочее)
- [ ] Напоминание перезагрузить фильтр в игре (Options → UI) после патча

### Этап 5 — Оверлей + хоткеи
- [ ] `ui/overlay.py`: прозрачное окно поверх игры
  - [ ] статус (вкл/выкл), активный профиль, кол-во целей в кадре
  - [ ] (опц.) визуализация радиуса и найденных точек
- [ ] Хоткеи: вкл/выкл, переключение профилей, калибровка

### Этап 6 — Профили и доп. автоматика
- [ ] Профили под локации/билды (свои цвета, радиус, приоритеты)
- [ ] (опц.) `core/automation.py`: авто-фласки/скиллы по таймеру

### Этап 7 — Калибровка и устойчивость
- [ ] Привязка к разрешению/масштабу UI PoE2
- [ ] Мастер калибровки цвета (кликнул по образцу → подобрал HSV-порог)
- [ ] (опц.) OCR-фолбэк для чтения названий предметов

## 4. Конфиг (черновик default.yaml)
```yaml
game:
  window_title: "Path of Exile 2"
capture:
  backend: dxcam        # dxcam | mss
  target_fps: 60
filter:
  path: "C:/Users/OLD/Documents/My Games/Path of Exile 2/Vladislav Yoshi Perfection.filter"
  marker_rgb: [255, 0, 254]         # уникальный фон-маркер override-блока
  categories:                        # что авто-собирать (override перекрасит в marker)
    - currency
    - fragments
    - waystones
vision:
  marker_hsv_low:  [149, 200, 200]  # HSV-окно вокруг marker_rgb (подбирается калибровкой)
  marker_hsv_high: [151, 255, 255]
  min_blob_area: 12
loot:
  pickup_radius_px: 400
  click_cooldown_ms: 90
  randomize_delay_ms: [20, 70]
  mode: hold            # hold | toggle | single
hotkeys:
  toggle: "f8"
  pickup: "space"
  calibrate: "f9"
  quit: "f12"
overlay:
  enabled: true
  show_radius: true
```
